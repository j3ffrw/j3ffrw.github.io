SATA port mapping in linux

commands to be used.

lshal

sysfs of interest:

/sys/block/sdx

/sys/class/scsi_disk

/sys/class/scsi_host/

lshal | less

find the udi block with the following hints.

info.product = SATA IDE Controller or SATA AHCI Controller

info.linux.driver = ‘ata_piix’ or ‘ahci’

on that udi block take note of the value of pci.linux.sysfs_path

this points to the device root symlink.

try to list contents the contents of that path

ls -alh /sys/devices/pci0000:00/0000:00:1f.2

you will find directories name host0 and host1, probably host2 and so forth. This are the host controller

hostx is also accessible from /sys/class/scsi_host/

to check where a particular hardrive is connected.

ls -alh /sys/block/sdx

ex. ls -alh /sys/block/sdb

/sys/block/sdb -> ../devices/pci0000:00/0000:00:1f.2/host1/target1:0:0/1:0:0:0/block/sdb

host0/target0:0:0/0:0:0:0 maps to SATAII0 sda sda

host0/target0:0:1/0:0:1:0 maps to SATAII1 sdb sda

host1/target1:0:0/1:0:0:0 maps to SATAII2 sdc sdb

host1/target1:0:1/1:0:1:0 maps to SATAII3 sdd sdc sbb
